# Google API セットアップガイド

このアプリケーションを動作させるためには、Google Cloud Consoleでプロジェクトを作成し、Google Drive APIとGoogle Picker APIを有効化する必要があります。また、GitHub Pagesなどの静的ホスティングで動作させるため、クライアントIDとAPIキーを取得します。

## 手順

1. **Google Cloud Consoleへのアクセス**
   - [Google Cloud Console](https://console.cloud.google.com/) にアクセスし、Googleアカウントでログインします。

2. **新しいプロジェクトの作成**
   - 画面左上のプロジェクトドロップダウンをクリックし、「新しいプロジェクト」を選択します。
   - 任意のプロジェクト名（例: "Drive MD Viewer"）を入力し、「作成」をクリックします。

3. **APIの有効化**
   - 左側のナビゲーションメニューから「APIとサービス」 > 「ライブラリ」を選択します。
   - 以下の2つのAPIを検索し、それぞれ「有効にする」をクリックします。
     - **Google Drive API**
     - **Google Picker API**

4. **OAuth同意画面の設定**
   - 左側のメニューから「APIとサービス」 > 「OAuth同意画面」を選択します。
   - 「User Type」を「外部」に設定し、「作成」をクリックします。
   - 必須項目（アプリ名、ユーザーサポートメール、デベロッパーの連絡先情報）を入力し、「保存して次へ」をクリックします。
   - スコープ画面で「スコープを追加または削除」をクリックし、以下のスコープを追加します。
     - `https://www.googleapis.com/auth/drive.readonly`
   - 「保存して次へ」をクリックし、テストユーザー画面に進みます。
   - （アプリが公開前の場合）自分のGoogleアカウントのメールアドレスをテストユーザーとして追加します。

5. **認証情報の作成（OAuth クライアント ID）**
   - 左側のメニューから「認証情報」を選択します。
   - 「認証情報を作成」 > 「OAuth クライアント ID」をクリックします。
   - アプリケーションの種類として「ウェブ アプリケーション」を選択します。
   - 「承認済みの JavaScript 生成元（Authorized JavaScript origins）」に、アプリケーションをホストするURLを追加します。
     - 開発用: `http://localhost:3000`
     - 本番用（GitHub Pages等）: `https://<あなたのユーザー名>.github.io`
   - 「作成」をクリックすると、**クライアントID** が表示されます。これをコピーして控えておきます。

6. **認証情報の作成（API キー）**
   - 再び「認証情報を作成」 > 「API キー」をクリックします。
   - 生成された **API キー** をコピーして控えておきます。
   - （推奨）作成されたAPIキーの「APIキーを編集」を開き、「APIの制限」で「Google Drive API」と「Google Picker API」のみに制限することをお勧めします。

7. **アプリへの設定入力**
   - GitHub Pages などでホスティングされたアプリケーションにアクセスします。
   - 初回アクセス時に設定画面が表示されるので、手順5と6で取得した「クライアントID」と「APIキー」を入力して保存します。
   - この情報はブラウザの LocalStorage にのみ保存され、安全に扱われます。

これで、アプリケーションがGoogle Driveからファイルを読み込む準備が整いました。
